Présentation du projet

Ce projet a consisté à développer une application de covoiturage permettant aux utilisateurs de proposer ou de rechercher des trajets quotidiens. Le but est de favoriser le transport partagé tout en facilitant la mise en relation entre conducteurs et passagers.

Mode de fonctionnement interne

Le cœur du projet repose sur une base de données MySQL composée des tables suivantes :

  • utilisateurs : informations personnelles, rôle (conducteur/passager)
  • trajets : trajets proposés ou recherchés
  • correspondances : associations entre trajets compatibles
  • messages : système de messagerie entre utilisateurs
  • notifications : alertes automatiques

Chaque entité est reliée par des clés étrangères assurant la cohérence des données.

Description de la conception

Le modèle de base de données a été pensé pour assurer :

  • Une intégrité référentielle rigoureuse (via les contraintes FOREIGN KEY)
  • Un rôle défini pour chaque utilisateur
  • Une évolutivité permettant d’ajouter d’autres fonctionnalités (paiement, historique...)

Extrait du fichier bdd.sql :

CREATE TABLE utilisateurs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nom VARCHAR(100),
    email VARCHAR(255),
    mot_de_passe VARCHAR(255),
    role ENUM('conducteur', 'passager')
);

Instructions de déploiement

  1. Installer MySQL et un éditeur SQL (Workbench, DBeaver...)
  2. Créer la base de données avec la commande :
    3. CREATE DATABASE covoiturage_app;
  3. Exécuter le script bdd.sql pour créer toutes les tables
  4. Vérifier l’insertion de données de test (utilisateurs, trajets...)
  5. Tester des requêtes d’ajout, de jointure et de suppression

Manuel d'utilisation

  • L'utilisateur s’inscrit (insertion dans utilisateurs)
  • Il propose ou recherche un trajet (trajets)
  • Le système cherche une correspondance compatible
  • Une discussion s’ouvre automatiquement (messages)
  • Une notification est générée

Chaque action peut être simulée par une requête SQL.
